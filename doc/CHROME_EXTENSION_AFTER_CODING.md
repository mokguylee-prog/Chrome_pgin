# Chrome Extension 코드 작성 후 해야 할 일

이 문서는 "코드 작성이 끝난 뒤" 실제 배포/운영까지 진행하기 위한 실무 체크리스트입니다.

## 1) 로컬에서 확장 프로그램 로드

1. Chrome에서 `chrome://extensions` 접속
2. 우측 상단 `개발자 모드` 활성화
3. `압축해제된 확장 프로그램 로드` 클릭
4. 프로젝트 루트 폴더 선택
5. 확장 프로그램 카드에서 에러가 없는지 확인

## 2) 기본 동작 테스트

1. `popup` 열림/닫힘 정상 동작 확인
2. `options` 페이지 저장/불러오기 확인
3. `content script`가 의도한 사이트에서만 동작하는지 확인
4. `background service worker` 이벤트 처리 확인
5. 새로고침, 탭 이동, 브라우저 재시작 후에도 정상 동작하는지 확인

## 3) 에러/로그 점검

1. 확장 프로그램 카드의 `service worker` 검사(Inspect) 열기
2. Console 에러/경고 제거
3. 네트워크 요청 실패(401/403/429/500) 처리 확인
4. 메시지 통신(`sendMessage`) 실패 케이스 대응 확인

## 4) 권한 및 보안 점검

1. `manifest.json`의 권한을 최소화 (`permissions`, `host_permissions`)
2. 불필요한 전체 도메인 권한 제거
3. 사용자 데이터 저장 위치(`storage.local/sync`) 검토
4. 민감 정보(API 키/토큰) 하드코딩 여부 확인
5. 외부 스크립트/리소스 로딩 정책(CSP) 위반 여부 확인

## 5) UX/품질 점검

1. 아이콘(16/48/128) 누락 여부 확인
2. 팝업 UI의 좁은 화면(작은 높이/너비) 대응 확인
3. 에러 상황에서 사용자 안내 문구 표시 확인
4. 한국어/영어 등 문자열 오타 및 일관성 점검
5. 주요 플로우 수동 테스트 시나리오 문서화

## 6) 버전 및 릴리즈 준비

1. `manifest.json`의 `version` 업데이트
2. 변경사항 요약(릴리즈 노트) 작성
3. 배포용 폴더 정리(불필요 파일 제외)
4. 배포용 ZIP 생성 (루트에 `manifest.json`이 오도록 압축)

## 7) Chrome Web Store 등록 준비

1. 스토어 설명문(짧은 설명/상세 설명) 작성
2. 스크린샷, 아이콘, 프로모션 이미지 준비
3. 개인정보처리방침 URL 준비(데이터 수집 시 사실상 필수)
4. 데이터 사용 관련 질문(수집/공유/목적) 답변 초안 준비
5. 테스트 계정이 필요한 경우 심사용 계정 정보 준비

## 8) 제출 전 최종 체크

1. 설치 직후 첫 실행 경험(온보딩) 확인
2. 확장 프로그램 비활성/재활성 시 복구 동작 확인
3. 오프라인/저속 네트워크 환경에서 동작 확인
4. 크래시를 유발할 수 있는 예외 처리 여부 확인
5. 이전 버전 사용자 업데이트 시 마이그레이션 확인

## 9) 배포 후 운영

1. 사용자 리뷰/문의 채널 모니터링
2. 에러 리포트 및 재현 케이스 수집
3. 권한 변경이 포함된 업데이트는 변경 이유를 릴리즈 노트에 명확히 기재
4. 작은 수정도 버전 관리 규칙에 따라 배포
5. 정기적으로 Chrome 정책 변경 사항 확인

4. 스토어 제출 자료 준비
5. 제출 후 모니터링 및 핫픽스 대응
